User Stories & Acceptance Criteria

Epic 1: Express Checkout

Story 1.1: One-Click Purchase for Returning Users

As a returning Tamara user on mobile,
I want to complete my purchase with one tap,
So that I can buy items quickly during flash sales.

Acceptance Criteria:

  • User taps "Buy Now" and purchase completes in ≤ 3 seconds
  • Shipping address auto-filled from last order (85% accuracy)
  • Payment method pre-selected (last used card/BNPL plan)
  • OTP skipped if device recognized (biometric fallback)
  • Order confirmation shown with delivery ETA

Story 1.2: Guest Checkout for New Users

As a first-time shopper,
I want to checkout without creating an account,
So that I can complete my purchase faster.

Acceptance Criteria:

  • Guest flow requires only: phone, email, shipping address
  • Post-purchase account creation optional (email link)
  • Guest orders trackable via SMS link for 30 days

Epic 2: SmartPay AI

Story 2.1: Instant Credit Decision

As a shopper with no credit history,
I want to receive an instant approval decision,
So that I don't abandon my cart waiting.

Acceptance Criteria:

  • Credit decision rendered in < 800ms (P95)
  • If approved: credit limit and installment plan displayed
  • If declined: alternative payment methods suggested (debit, cash)
  • Soft credit check only (no SIMAH hard inquiry unless user proceeds)
